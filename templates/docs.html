How CleanSweep Works

Notice: This cloud-hosted site at Vercel is a read-only telemetry preview. For safety, actual file scans, allowlist queries, and directory cleanups are disabled in deployment.

CleanSweep is an offline-first system optimizer. It scans temporary directories, cache folders, and package manager locations in your local home directory to identify space that can be reclaimed.

The scanning and deletion are handled completely by a lightweight local Python backend (Flask). The visual dashboard is a standard client-side HTML interface that retrieves analytics dynamically from local API endpoints.

Architecture & Cloud Boundaries

CleanSweep uses a hybrid architecture to balance accessibility with strict local privacy boundaries:

  • Hosted Preview Mode (Vercel): The public site at cleansweep-blush.vercel.app serves as a **read-only static mockup and documentation center**. For safety, folder scanning and deletion API endpoints are completely disabled in Vercel previews. It cannot access your local telemetry or files.
  • Local Client (Homebrew/Source): The active desktop utility runs 100% locally on your machine. The Flask server serves local assets only to your browser client loop at localhost:5051.

No Remote Tracking

There is no database or network connection between your local client installation and the hosted cloud preview. They are fully decoupled environments. Your file paths and local telemetry never communicate externally.

Privacy & Threat Model

100% Offline by Design

CleanSweep executes entirely on your host CPU. It makes zero external telemetry reports, tracking metrics, or analytics calls. Your local path folders, usernames, and metrics never leave your computer.

Because CleanSweep does not use a cloud agent, it runs with minimal threat overhead. The source code is open, auditable, and carries no third-party tracking scripts.

Safety Gates & Verification Code

To prevent accidental file deletion or damage to essential system/user paths, CleanSweep implements a strict allowlist matching filter:

  • Directory Boundary Auditing: Every deletion candidate path is resolved using Path.resolve() to remove symlink masking or parent directory traversal hacks (e.g. ../../).
  • Allowlisted Cache Roots: Deletion is strictly constrained to specific folders defined in your config.py (e.g., specific user browser cache subfolders, Pip cache subdirectories, and npm cache folders).
  • Explicit Exclusions / Blocklist Guard: Even if a path resides inside an allowlisted folder, CleanSweep explicitly skips files matching custom sensitive patterns:
    • System directories: /System, /usr, /etc, /bin, /sbin, /Library/SystemProfiler
    • Security/Access keys: ~/.ssh, ~/.aws, .env configurations
    • Hidden revision folders: .git, .github pipelines

What gets Scanned & Cleaned

Only specific non-critical temporary folders are touched during scans:

1. Browser Cache Locations

  • Google Chrome caches: ~/Library/Caches/Google/Chrome
  • Safari caches: ~/Library/Caches/com.apple.Safari

2. Package Managers Caches

  • npm cache: ~/.npm/_cacache
  • Cargo (Rust) cache: ~/.cargo/registry/cache
  • Pip (Python) cache: ~/Library/Caches/pip
  • Conda environment cache: ~/.conda/pkgs

3. OS Temp Files

  • System temp directories (/tmp or user-level environment temp folders).
  • Application log files under ~/Library/Logs.

What is NEVER Deleted

CleanSweep enforces strict exclusion rules to safeguard user documents. The following paths and files are completely skipped:

  • Personal Documents: Folders like ~/Documents, ~/Desktop, ~/Downloads, ~/Pictures, ~/Music, and ~/Movies are never targetable by cleaners.
  • Project Source Code: Code structures, project workspaces, and repository checkouts are excluded.
  • System Roots: System files, binary environments (/usr/bin, /bin), and operating system core assets are locked.

Platform Support & Limitations

CleanSweep adapts its scanning paths dynamically depending on the host OS. Support levels are defined as follows:

Additional Notes & Caveats

  • macOS Gatekeeper: Standalone compiled packages (.app) are unsigned. macOS users are recommended to install warning-free via the Homebrew HEAD formula to bypass quarantine blocks.
  • macOS Telemetry: Native system stats require the psutil Python hook.
